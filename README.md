# 🐧 Linux 프로세스 관리 명령어 조사 (과제 #2)

![Linux](https://img.shields.io/badge/Linux-FCC624?style=for-the-badge&logo=linux&logoColor=black)
![Git](https://img.shields.io/badge/git-%23F05033.svg?style=for-the-badge&logo=git&logoColor=white)

## 🎯 개요
이 문서는 **'오픈소스SW개론'** 과제 #2로, Linux의 주요 프로세스 및 작업 관리 명령어인  
`top`, `ps`, `jobs`, `kill` 4가지 명령어에 대해 조사한 내용을 정리한 문서입니다.

각 명령어는 **서버 운영**, **백엔드 개발**, **시스템 관리**, **프로세스 디버깅** 등에서 반드시 알아야 하는 핵심 명령어입니다.  
이 문서에서는 기본 사용법뿐 아니라, **동작 원리**, **실제 사용 시나리오**, **옵션별 상세 설명**을 포함해 심층적으로 정리했습니다.

---

---

# 1. 📈 top — (시스템 실시간 모니터링)

`top` 명령어는 Linux 시스템의 **실시간 성능 상태를 모니터링**하는 대표적인 도구입니다.  
프로세스별 CPU 사용량, 메모리 사용량, 실행 시간, 우선순위(NICE), Load Average(부하율) 등을  
**1~3초 간격으로 갱신하여** 보여줍니다.

---

## 📋 주요 기능

### ✔ 1) 시스템 전반 상태 확인  
top의 상단 영역에서는 다음 정보를 실시간으로 표시합니다:
- 시스템 시간 / 업타임(uptime)
- 로그인한 사용자 수
- Load Average (1, 5, 15분 평균 부하)
- 전체/사용 가능한 작업(Task)의 개수
- CPU 사용 비율 (us, sy, id, wa 등)
- 메모리·스왑 사용량

### ✔ 2) 프로세스 목록 실시간 모니터링  
top 하단에는 개별 프로세스가 표시되며 다음 정보를 확인할 수 있습니다:
- PID
- USER
- CPU% / MEM%
- TIME+ (프로세스가 사용한 총 CPU 시간)
- COMMAND (실행 중인 명령)

---

## 🔧 유용한 top 단축키

| 키 | 기능 |
|----|-----|
| `P` | CPU 사용률 기준 정렬 |
| `M` | 메모리 사용률 기준 정렬 |
| `k` | 프로세스 종료(kill 요청) |
| `h` | 도움말 표시 |
| `q` | 종료 |

---

## 💻 사용 예시

```bash
# 기본 실행
top
# 종료 시 'q' 키 입력
```

## 2. 📸 ps (프로세스 스냅샷 확인)

`ps` 명령어는 명령어를 실행한 **특정 시점**의 프로세스 상태를 스냅샷처럼 **정적으로** 보여줍니다.

### ⭐ 자주 사용되는 옵션
| 옵션 | 설명 | 비고 |
| :---: | :--- | :--- |
| `-e` | 현재 실행 중인 **모든 프로세스** 표시 | |
| `-f` | **Full Format**으로 상세 정보 표시 | UID, PID, PPID 등 포함 |
| `aux` | 모든 사용자의 프로세스를 상세하게 표시 | **BSD 스타일** (가장 많이 사용됨) |

### 🔍 활용 예시
```bash
# 모든 프로세스 상세 조회
ps aux

# 특정 프로세스(예: python) 검색
ps -ef | grep python
```

## 3. 💼 jobs (백그라운드 작업 관리)

`jobs` 명령어는 현재 셸 세션에서 **백그라운드(&)**로 실행 중이거나 중지된 작업의 목록을 관리합니다.

### 🔄 작업 제어 명령어
| 명령어 | 설명 | 예시 |
| :--- | :--- | :--- |
| `&` | 명령어를 **백그라운드**에서 실행 | `sleep 300 &` |
| `jobs` | 백그라운드 작업 **목록 확인** | `jobs` |
| `fg` | 작업을 **포그라운드(Foreground)**로 전환 | `fg %1` |
| `bg` | 멈춘 작업을 **백그라운드(Background)**로 전환 | `bg %1` |

### 💻 실습 예시
```bash
# 300초 대기 작업을 백그라운드로 실행
sleep 300 &

# 작업 목록 확인 ([1] Running sleep 300 &)
jobs
```

## 4. 🔪 kill (프로세스 종료 및 제어)

`kill` 명령어는 PID(Process ID)를 사용하여 특정 프로세스에 **시그널(Signal)을 전송**하여 종료하거나 제어합니다.

### 🚨 주요 시그널 (Signal)
| 번호 | 이름 | 설명 | 특징 |
| :---: | :---: | :--- | :--- |
| **1** | `SIGHUP` | 재시작 요청 | 설정 파일 재로드 시 사용 |
| **9** | `SIGKILL` | **강제 종료** | 프로세스가 무시할 수 없음 (즉시 종료) |
| **15** | `SIGTERM` | **정상 종료** | 종료 전 정리 작업 수행 가능 (권장) |

### 💻 사용 예시
```bash
# PID 12345 프로세스에 정상 종료 요청 (권장)
kill -15 12345

# PID 54321 프로세스 강제 종료 (응답 없을 시)
kill -9 54321
```
