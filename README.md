# 🐧 Linux 프로세스 관리 명령어 조사 (과제 #2)

## 🎯 개요
이 문서는 '오픈소스SW개론' 과제 #2로, Linux의 주요 프로세스 및 작업 관리 명령어인 `top`, `ps`, `jobs`, `kill`에 대해 조사한 결과를 정리했습니다.

---

## 1. top 명령어
실시간으로 시스템의 **프로세스 활동**과 **성능**을 모니터링하는 명령어입니다.

* **주요 기능**: CPU, 메모리 사용량, 실행 시간 등 시스템 자원 사용 현황을 동적으로 보여줍니다.
* **사용 예시**:
    ```bash
    top
    ```
* **종료**: `q` 키를 누르면 종료됩니다.

## 2. ps 명령어
현재 시스템에서 **실행 중인 프로세스의 스냅샷**을 보여주는 명령어입니다. `top`과는 달리 정적인 정보(특정 시점의 상태)를 제공합니다.

* **주요 옵션**:
    * `-e`: 모든 프로세스 표시
    * `-f`: 전체 형식(Full format)으로 상세 정보 표시
    * `aux`: 모든 사용자의 프로세스를 터미널 및 상세 정보와 함께 표시 (BSD 스타일)
* **사용 예시**:
    ```bash
    ps aux
    ```
    ```bash
    ps -ef | grep <프로세스 이름>  # 특정 프로세스 검색
    ```

## 3. jobs 명령어
현재 셸에서 실행 중이거나 중지된 **백그라운드 작업(Job)**의 목록을 확인하는 명령어입니다.

* **주요 기능**: 작업 번호, 상태(Running, Stopped), 실행 명령을 보여줍니다.
* **작업 실행 예시**:
    ```bash
    sleep 300 &   # 300초 동안 대기하는 작업을 백그라운드(&)로 실행
    jobs          # 백그라운드 작업 목록 확인
    ```
* **작업 제어**:
    * `fg %1`: 작업 번호 1을 포그라운드(Foreground)로 전환
    * `bg %1`: 중지된 작업 번호 1을 백그라운드(Background)로 전환

## 4. kill 명령어
지정된 **프로세스에 시그널을 보내어** 종료하거나 제어하는 명령어입니다.

* **사용 형식**: `kill [시그널] <PID>`
* **주요 시그널**:
    | 시그널 번호 | 시그널 이름 | 설명 |
    | :---: | :---: | :--- |
    | 1 | SIGHUP | 터미널 연결 해제 또는 설정 파일 재로드 |
    | 9 | SIGKILL | **강제 종료** (프로세스가 무시할 수 없음) |
    | 15 | SIGTERM | **정상 종료 요청** (프로세스가 종료 전에 정리 작업 가능) |
* **사용 예시**:
    ```bash
    # PID 12345인 프로세스에 정상 종료 요청
    kill 15 12345
    ```
    ```bash
    # PID 54321인 프로세스를 강제 종료
    kill -9 54321
    ```
