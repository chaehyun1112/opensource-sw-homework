# 1. 📈 top — (시스템 실시간 모니터링)

`top` 명령어는 Linux 시스템의 **실시간 성능 상태를 모니터링**하는 대표적인 도구입니다.  
프로세스별 CPU 사용량, 메모리 사용량, 실행 시간, 우선순위(NICE), Load Average(부하율) 등을  
**1~3초 간격으로 갱신하여** 보여줍니다.

---

## 📋 주요 기능

### ✔ 1) 시스템 전반 상태 확인  
top의 상단 영역에서는 다음 정보를 실시간으로 표시합니다:
- 시스템 시간 / 업타임(uptime)
- 로그인한 사용자 수
- Load Average (1, 5, 15분 평균 부하)
- 전체/사용 가능한 작업(Task)의 개수
- CPU 사용 비율 (us, sy, id, wa 등)
- 메모리·스왑 사용량

### ✔ 2) 프로세스 목록 실시간 모니터링  
top 하단에는 개별 프로세스가 표시되며 다음 정보를 확인할 수 있습니다:
- PID
- USER
- CPU% / MEM%
- TIME+ (프로세스가 사용한 총 CPU 시간)
- COMMAND (실행 중인 명령)

---

## 🔧 유용한 top 단축키

| 키 | 기능 |
|----|-----|
| `P` | CPU 사용률 기준 정렬 |
| `M` | 메모리 사용률 기준 정렬 |
| `k` | 프로세스 종료(kill 요청) |
| `h` | 도움말 표시 |
| `q` | 종료 |

---

## 💻 사용 예시

```bash
# 기본 실행
top
# 종료 시 'q' 키 입력
```

## 2. 📸 ps (프로세스 스냅샷 확인)

`ps` 명령어는 시스템의 현재 프로세스 상태를 “정적인 스냅샷” 형태로 보여주는 명령어입니다.  
`top`이 실시간 갱신을 제공하는 반면, `ps`는 명령을 입력한 순간의 상태만 출력한다는 점이 핵심입니다.

📌 즉
1) top = 실시간 변화 표시
2) ps = 정적인 정보 조회(필터링 가능)

---

### ⭐ 주요 특징

- **정적 출력**: 실행 시점의 상태만 보여줌  
- **필터링 및 옵션 활용 용이**  
- **백그라운드/고아 프로세스 추적에 유용함**  
- **스크립트 자동화에 자주 사용됨** (파싱하기 쉬움)

---

### ⭐ 자주 사용되는 옵션
| 옵션 | 설명 | 비고 |
|------|------|------|
| `-e` | 모든 프로세스를 출력 | systemd, kernel task 포함 |
| `-f` | Full Format 출력 | PPID, UID, CMD 등 표시 |
| `aux` | 모든 사용자 + 상세정보(BSD타입) | 가장 많이 사용됨 |
| `-ef` | Linux 표준 형식(full format) | PID 추적 시 많이 사용 |

---

### 📋 츌력 예시(ps aux)
```bash
# 모든 프로세스 상세 조회
ps aux
```

### ⭐ 츌력 컬럼 의미
| 컬럼 | 설명 |
|------|------|
| USER | 실행 중인 사용자 |
| PID | 프로세스 ID |
| %CPU | CPU 점유율 |
| %MEM | 메모리 점유율 |
| VSZ | 가상 메모리 사용량 |
| RSS | 실제 메모리 사용량 |
| STAT | 프로세스 상태(R, S, T, Z 등) |
| START | 시작 시각 |
| TIME | 사용한 CPU 시간 |
| COMMAND | 실행된 명령 |

---

### 🔍 특정 프로세스(예: python) 검색
```bash
ps -ef | grep python
```

## 3. 💼 jobs (백그라운드 작업 관리)

`jobs` 명령어는 현재 셸 세션에서 **백그라운드(&)**로 실행 중이거나 중지된 작업의 목록을 관리합니다.

### 🔄 작업 제어 명령어
| 명령어 | 설명 | 예시 |
| :--- | :--- | :--- |
| `&` | 명령어를 **백그라운드**에서 실행 | `sleep 300 &` |
| `jobs` | 백그라운드 작업 **목록 확인** | `jobs` |
| `fg` | 작업을 **포그라운드(Foreground)**로 전환 | `fg %1` |
| `bg` | 멈춘 작업을 **백그라운드(Background)**로 전환 | `bg %1` |

### 💻 실습 예시
```bash
# 300초 대기 작업을 백그라운드로 실행
sleep 300 &

# 작업 목록 확인 ([1] Running sleep 300 &)
jobs
```

## 4. 🔪 kill (프로세스 종료 및 제어)

`kill` 명령어는 PID(Process ID)를 사용하여 특정 프로세스에 **시그널(Signal)을 전송**하여 종료하거나 제어합니다.

### 🚨 주요 시그널 (Signal)
| 번호 | 이름 | 설명 | 특징 |
| :---: | :---: | :--- | :--- |
| **1** | `SIGHUP` | 재시작 요청 | 설정 파일 재로드 시 사용 |
| **9** | `SIGKILL` | **강제 종료** | 프로세스가 무시할 수 없음 (즉시 종료) |
| **15** | `SIGTERM` | **정상 종료** | 종료 전 정리 작업 수행 가능 (권장) |

### 💻 사용 예시
```bash
# PID 12345 프로세스에 정상 종료 요청 (권장)
kill -15 12345

# PID 54321 프로세스 강제 종료 (응답 없을 시)
kill -9 54321
```
